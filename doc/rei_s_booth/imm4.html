Conversion Mode

IME has two status flags which indicate the current mode of converting the input string. One is "Conversion Mode", the other is "Sentence Mode".

The conversion mode decides a kind of letters for input string, such as hiragana/katakana/alphanumeric, full-shape/half-shape, roman/kana/code. Some modes can be combined, for example, "katakana + full-shape + kana".

The sentence mode indicates how to analyze the sentence to be converted. For example, the phrase will be predicted, the name of a person/place will have priority, a spoken language will take precedence or no conversion will be performed. This property is usually decided by the user and rarely changed by applications.

To know when the user changes the conversion mode, check for the WM_IME_NOTIFY message with IMN_SETCONVERSIONMODE value, which will be sent to the application window.

Here FAQs come:

How to change the conversion mode of the IME?

Your application may want to set the conversion mode of the IME as "full-shape hiragana" on one edit control, "half-shape katakana" on another edit control.

Both the conversion and sentence modes are described in a double-word value. The conversion mode is specified by a combination of bit flags prefixed by IME_CMODE_xxx, while the sentence mode is one of the values prefixed by IME_SMODE_xxx. See your InfoView or imm32.h header file for the definitions.

To retrieve and set the current IME status, use ImmGetConversionStatus and ImmSetConversionStatus functions.

How to activate the software keyboard provided by the IME?

If IME supports software keyboard and/or character-code input, they can (should) also be activated by ImmSetConversionStatus. Including IME_CMODE_SOFTKBD or IME_CMODE_CHARCODE will open IME window which allows you to input undetermined characters by clicking mouse or typing the kanji-code. (The result dependends on the implementation of each IME.)
