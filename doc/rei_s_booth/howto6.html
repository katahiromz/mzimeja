How to Hide IME's Status Window

Some IMEs always show their status windows even if closed or disabled, some do only when opened. We sometimes want to hide the IME's status window, because it might be an obstacle to the application window (if the window never uses the IME).

Note that status window is usually the only way to show the current conversion mode. If your program *always* hide it, users may be forced to put up with inconvenience. Do not interfere with the Window's general user interface.

As for Microsoft IMEs, MS-IME 97, MS-IME 98 and MS-IME 2000 have an user option to select when to show the status window. If the window is shown on the desktop even when the IME is closed, it may be the user's choise. MS-IME 95 does not have this option and always places the status window on the user's desktop. Most of other IMEs (in Japan) would display the status window only when the IME is opened.

After considerating these specs, you may still need to hide the status window. It can be done by WM_IME_CONTROL message with IMC_CLOSESTATUSWINDOW in wParam.
However, the status window will appear again at some triggers - for example, when the user changes the current context and be back to your context. So you may need to monitor WM_IME_NOTIFY message to know when the status window is shown, and hide it again in response.
