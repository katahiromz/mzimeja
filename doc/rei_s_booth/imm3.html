IME Status

IME status on taskbar
(on, off, disabled)

There are three types of IME status - on (opened), off (closed) and disabled. Looking at a pen icon on the taskbar notification area, you can see the current IME status of the current active window.
If the pen is red, IME is closed (but can be opened when needed). The red pen with a document indicates that IME is opened. And if the pen is grayed, IME is disabled - you can never open it.

If the IME is opened or closed by the user's operation, your application window will receive WM_IME_NOTIFY message with IMN_SETOPENSTATUS value. You can use ImmGetOpenStatus to know the current IME status.

How to open or close the IME?

Sometimes you may want to open IME automatically, for example, when the input focus moves to an edit control. Use ImmSetOpenStatus function for this purpose. A user can (re)open or close it anytime he want.

How to disable the IME?

Sometimes you may want to disable the IME, that is, to prevent the user from turning IME on. To implement this behavior, associate the window with NULL context by ImmAssociateContext. This function returns the previous hIMC. If you want to restore the hIMC after some process has been completed, you can use the return value of the function.

Win98
Windows 98 (and Windows 2000) has a new function called ImmDisableIME, which takes the thread ID as the only parameter. According to the description of the Platform SDK, it disables the IME for the specified thread or the current process.
