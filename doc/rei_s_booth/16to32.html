Shifting to IMM32

On the earlier version of Windows, the IME had been managed by the system module called WINNLS(32).
Still Win95/NT3.51 and higher have a low compatibility with the older IME interface, 32bit applications should not use them. Especially, if your application which is (re-)compiled for Win4 platforms is to handle the old IME messages, you will find that they won't be sent to your window.

The following table would be helpful when you re-write Win3 codes to Win4 native. (Please view with a browser which supports tables if possible.)
